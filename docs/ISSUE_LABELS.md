# Recommended GitHub Issue Labels

The following labels are recommended for the TCP/AI repository to standardize project management and issue tracking.

| Label Name | Description |
| :--- | :--- |
| `phase` | Corresponds to major project roadmap phases. |
| `tech-debt` | Internal structural cleanup or architectural improvements. |
| `tooling` | Changes or improvements to automated scripts and CLI tools. |
| `benchmark` | Methodology updates or empirical result reporting. |
| `corpus` | Additions, removals, or edits to the prompt-pair dataset. |
| `spec` | Refinements or clarifications to the canonical specification. |
| `ci` | Changes to GitHub Actions or automated validation logic. |
| `good-first-issue` | Accessible tasks suitable for new contributors. |
| `documentation` | Updates to README, ROADMAP, SPEC, or other docs. |

*Note: These labels should be created in the GitHub repository settings during the next governance step.*
