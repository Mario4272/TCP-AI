# Governance

TCP/AI is currently a draft specification maintained by the community. We use a lightweight, benevolent dictator for now (BDFL) model with strong community input, aiming to transition to a more formalized consensus model as the project grows.

## Decision Making Process
1. **Proposals**: All specification changes should be proposed via the `Specification Proposal` issue template.
2. **Discussion**: Community members will discuss the proposal, focusing on clarity, token efficiency, and impact on response quality.
3. **Consensus**: We aim for lazy consensus. If there are no major objections after a reasonable period, the proposal moves forward.
4. **Approval**: Core maintainers will provide final approval and merge the corresponding Pull Request.

## Contributions
We welcome contributions in the form of benchmarks, documentation improvements, and tool development. Please refer to `CONTRIBUTING.md` for guidelines.

## Code of Conduct
Please be respectful, collaborative, and constructive in all interactions within issues and pull requests.
